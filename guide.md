# Guide til webdev

## Tre hoved inndelinger:

### Header

- Logo
- Søkefelt (Vi lager visuelt ikke funksjonabelt)
- Innlogging (Vi lager visuelt ikke funksjonabelt)
- Navigasjon = tekst eller bilde ut i fra målgruppen
- SoMe = mer tilgjengelig i header enn footer

### Main

- Viktigest å få med seg øverst = hero
- Hovedmålet med siden = om deg/bedrift, fremvisning av produkt/prosjekt
- Påminnelse om tidligere innhold om lang side
- Main er hovedsaklig landingssidene til navigasjon

### Footer

- Logo
- Videre navigasjon for mer informasjon
- SoMe = kun i footer kan være for mye "scrolling"
- Kontakt med adresse for besøk
- Kreditasjon
- Personværserklæring (For mer offesielle sider)

## Bruker reisen: Hvem, hva og hvordan

### Hvem:

- Målgrupper:
  - Ung: Video/audio, mange farger, avanserte font familier, synlig SoMe, MPA (Multi page application)
  - Eldre: Ren tekst sammen med bilder, synlig kontakt, solid større font familie, SPA (Single page application)

### Hva:

- Hva handler siden om:
  - Informativ nettside: statisk vs bevegelig
  - Butikk nettside: tydelig hva som selges
  - Portfolio nettside: forfattaren vs arbeidet
  - Forum nettside: statisk spørsmål/svar vs beveglig SoMe

## Hvorfor:

- Hva skal siden oppnå:
  - Informativ nettside: MPA vs SPA, tekst vs video, synlig SoMe
  - Butikk nettside: MPA vs SPA, tekst og bilder, stor- vs liten tekst, synlig kontakt
  - Portfolio nettside: tekst vs bilde, MPA vs SPA, synlig SoMe og kontakt
  - Forum nettside: tekst, bilde og video, MPA
